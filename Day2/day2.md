## 🧠 What I Worked On

Today I learned the difference between a **lazy prompt** and an **engineered prompt**, and saw firsthand how much the quality of a prompt changes the quality of an AI's response.

I used Claude to:
1. Generate a response/output from a vague, lazy prompt
2. Generate a response/output from a detailed, engineered prompt
3. Compare both outputs for explanation quality, structure, examples, and design
4. Create a LinkedIn-ready graphic explaining the concept of Prompt Engineering

---

## 💡 Key Learning

> A vague prompt gets a vague answer. A specific, well-structured prompt — one that includes role, context, audience, and format — gets a focused, professional, ready-to-use answer.

Prompt Engineering isn't about "tricking" the AI — it's about communicating clearly, the same way clear instructions help a human collaborator do better work.

## 📊 Weak vs Engineered — Example Comparison

| | Weak Prompt | Engineered Prompt |
|---|---|---|
| **Prompt** | "Tell me about marketing." | "Act as a marketing expert. Explain 3 beginner social media strategies for a small bakery, with an example each." |
| **Output** | Generic, unfocused, vague | Structured, specific, usable |
| **Why** | No role, no audience, no format | Clear role + audience + count + format |

---

## ✅ Three Benefits of Better Prompts

1. **Saves Time** — fewer retries, faster first-try results
2. **More Accurate** — specific prompts reduce vague or irrelevant answers
3. **Professional Output** — matches the tone and depth actually needed for real work
